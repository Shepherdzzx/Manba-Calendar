# Manba Alert

一个面向高频日程记录场景的语音日历助手。目标是通过语音交互帮助用户快速添加、查看、删除日程，并逐步扩展到提醒等能力。

## 当前状态

当前仓库处于**规划阶段**，尚无业务代码或可执行工程。

仓库现有内容：

- `README.md` -- 项目说明
- `.gitignore` -- 通用忽略规则
- `.claude/agents/*.md` -- Claude agent 配置文件（辅助开发流程，非项目业务代码）

Flutter 工程、ASR 语音识别、大模型集成等业务代码尚未初始化。

## 项目方向

核心交互链路：

```
用户语音 → ASR 转写文本 → 大模型解析为结构化 JSON → 校验后执行日历操作
```

第一阶段聚焦三个能力：

1. 语音添加事件（如"明天下午三点和张三开会"）
2. 语音查看日程（如"今天还有什么安排"）
3. 语音删除事件（如"取消今晚七点的健身"）

技术方案概览：

- **客户端**: Flutter Android 应用，负责录音与确认交互
- **语音识别**: ASR 模型将语音转写为文本
- **语义解析**: 大模型基于 prompt 将文本解析为固定格式 JSON 指令
- **校验执行**: 应用端校验 JSON 字段后执行日历操作，高风险操作增加确认步骤

## 开发说明

项目采用基于 PR 的分步开发方式：

- 每个 PR 只实现或修改单一功能
- PR 标题与描述需包含：标题、功能描述、实现思路、测试方式
- PR 合并后主分支应保持可运行状态

建议开发顺序：

1. 完成基础项目初始化和日历事件数据结构
2. 完成新增/查询/删除的文本流程
3. 完成 Flutter 基础页面、录音入口与权限处理
4. 接入 ASR 模型，实现语音转文字
5. 接入 Prompt + 大模型输出 JSON 指令
6. 增加 JSON 校验、执行确认与提醒能力优化
