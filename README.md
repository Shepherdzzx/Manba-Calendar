# Manba Alert

一个面向高频日程记录场景的语音日历助手。目标是通过语音交互帮助用户快速添加、查看、删除日程，并逐步扩展到提醒等能力。

## 当前状态

当前仓库已经完成 **Flutter MVP 骨架 + Go 本地解析 / 执行工具链** 两部分基础能力，不再处于纯规划阶段。

当前已落地内容：

- **Flutter 应用骨架**：包含首页日历、事件新增 / 编辑 / 软删除、语音输入入口、解析结果确认卡片、设置页与主题切换。
- **Go 本地解析器**：支持 `create_event` / `query_events` / `delete_event` 三类 intent，以及今天 / 明天 / 后天 / 周几 / 下周几 / 上午 / 下午 / 晚上 / 几点 / 几点半等高频中文时间表达。
- **Go 执行器与 CLI**：支持事件新增、查询、软删除、JSON 文件持久化，以及删除候选冲突时的确认流程。
- **协议与示例文档**：已提供 JSON 协议规范、本地解析器设计说明与语音命令示例。

当前仍未完成的部分：

- 真实麦克风录音与 ASR 集成
- Flutter 端对接真实解析器结果，替换当前 stub 语音服务
- 事件持久化与本地提醒通知能力

## 项目方向

核心交互链路：

```text
用户语音 → ASR 转写文本 → 端侧本地解析器解析为结构化 JSON → 校验后执行日历操作
```

第一阶段聚焦三个能力：

1. 语音添加事件（如“明天下午三点和张三开会”）
2. 语音查看日程（如“今天还有什么安排”）
3. 语音删除事件（如“取消今晚七点的健身”）

技术方案概览：

- **客户端**: Flutter 应用，负责日历展示、录音入口、确认交互与主题切换
- **语音识别**: 计划接入 Moonshine 等端侧 ASR 模型将中文语音转写为文本
- **语义解析**: 当前采用本地规则解析器，将 ASR 文本转换为固定格式 JSON 指令
- **校验执行**: 应用侧校验 JSON 字段后执行日历操作，高风险操作增加确认步骤

## 当前实现概览

### Flutter MVP

- 日历首页：点击日期查看当天事件
- 事件编辑器：支持新增、编辑、软删除
- 语音交互壳：按住识别、松开解析，并展示解析结果确认卡片
- 主题设置：内置紫金（默认）、青瓷、珊瑚夜三套配色
- 基础测试：覆盖 Provider 状态与首页组件显示

### Go 本地能力

- 本地解析器：中文自然语言 → 结构化 JSON 指令
- 执行器：根据结构化指令执行增 / 查 / 删
- CLI 调试工具：可独立验证解析与执行链路

## 开发说明

项目采用基于 PR 的分步开发方式：

- 每个 PR 只实现或修改单一功能
- PR 标题与描述需包含：标题、功能描述、实现思路、测试方式
- PR 合并后主分支应保持可运行状态

当前建议继续按以下方向拆分后续 PR：

1. 接入真实麦克风录音与 ASR
2. 打通 Flutter 与本地解析器 / 执行器的真实链路
3. 增加事件持久化
4. 增加提醒通知能力

## 本地开发

### 环境要求

- Flutter SDK（与 `pubspec.yaml` 中声明的 Dart SDK 兼容）
- Go（用于本地解析器 / CLI 调试）

### Flutter

```bash
flutter pub get
flutter test
flutter run
```

### Go 测试与调试

```bash
go test ./...
go run ./cmd/parser-cli --demo
go run ./cmd/executor-cli "明天下午三点和张三开会"
```
